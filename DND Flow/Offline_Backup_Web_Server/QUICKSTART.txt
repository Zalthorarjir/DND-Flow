ONLINE WEB SERVER - QUICK START
================================

1) Configure OAuth
   - Edit Online_Web_Server/.env
   - Set DISCORD_CLIENT_ID, DISCORD_CLIENT_SECRET, DISCORD_REDIRECT_URI
   - Ensure redirect URI exactly matches Discord developer portal

2) Install dependencies
   - pip install -r requirements.txt

3) Start server
   - python app.py
   - or run start_online_web_server_stack.bat (with Caddy flow)

4) Open browser
   - PUBLIC_BASE_URL from .env
   - Sign in with Discord OAuth

Common checks
-------------

- If external access fails:
  - Verify ONLINE_WEB_SERVER_HOST=0.0.0.0
  - Verify ONLINE_WEB_SERVER_PORT is forwarded/reachable
  - Verify PUBLIC_BASE_URL and DISCORD_REDIRECT_URI are the same public host

- If HTTPS errors appear:
  - Set ONLINE_WEB_SERVER_SSL_CERT / ONLINE_WEB_SERVER_SSL_KEY for direct TLS
  - or use reverse proxy and set TRUST_PROXY_HEADERS=True

- If bot does not reflect updates:
  - Restart Discord_Bot/main.py

Notes
-----

- Legacy USER_SERVER_* env keys are still accepted by app.py.
- Legacy start_user_server_stack.bat and stop_user_server_stack.bat now wrap the new script names.
